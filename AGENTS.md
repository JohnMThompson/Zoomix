# Repository Instructions

## Releases

- Do not create a release unless the user explicitly asks for one.
- Do not bump package versions, add release changelog entries, create or push
  version tags, or trigger a release workflow as part of an ordinary commit.
- Commits and pushes to the current branch do not require a new release.
- When a release is explicitly requested, keep `Cargo.toml`, `Cargo.lock`,
  `debian/changelog`, packaged documentation, and the Git tag version aligned.
